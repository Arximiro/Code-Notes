Flexbox

Container

  • You set display: flex in the parent container to affect all children
  • There is also inline-flex, but it's not used often. It just makes the flex container an inline element
  • The main axis is the way your flex box direction is flowing, either row or column
  • The cross axis is the opposite of the main

Flex Direction

  • The flex-direction defaults to row. There are also other choices: row-reverse, column and column-reverse
  • row-reverse just places the elements starting from the right side instead of the left
  • column is up and down and column-reverse is down to up

Flex Wrap

  • it defaults to nowrap, meaning items will not wrap to new lines based on viewport
  • this can cause items to get cut out completely as the viewport changes
  • setting to wrap makes items wrap to the next line of the main axis
  • wrap-reverse makes items do the same thing, just items from the other side of the container wrap first

Justify Content

  • justify-content determines how the objects inside the flexbox are displayed
  • the default is flex-start, but there are others: flex-end, center, space-between, space-around and space-evenly
  • flex-end pushes them to end the of the main axis which varies depending on your flex-direction
  • center centers items in the middle of the flexbox, so even space on all sides, but no spacing between items
  • space-between uses the entire container's space and gives each item equal spacing within the container
  • space-around gives items equal spacing on both sides of the element along the main axis
  • space-evenly gives items equal spacing along the main axis, taking into account space on the end

Align Items

  • align items determines how the items are aligned on the cross axis
  • align items defaults to stretch, so it will stretch all other items to match the largest
  • there is also flex-start, flex-end, center, and baseline
  • flex-start will align all items at the beginning of the cross axis
  • flex-end will align all items at the end of the cross axis
  • center will center all items on the cross axis relative to the largest element
  • baseline will align the content inside each element along the main axis

Align Content

  • keep in mind this property is used when you have multiple columns or rows on the cross axis
  • it's a way to manage the extra whitespace inside a parent container when there are multiple rows/columns
  • align content defaults to stretch, acts similar to space-between in execution
  • there is also flex-start, flex-end, center, space-between and space-around
  • flex-start has all content move to the top of the parent container, and all white space goes to the end
  • flex-end will push all content to the end, and have all extra white space to the start

Align Self

  • this property is used on the individual flex item
  • it aligns item to a different spot than all the others. ex. one item being flex-end while the rest are center
  • auto is the default, but there is stretch, flex-start, flex-end, center, and baseline
  • flex-start and flex-end align the item to either the start or end of the cross axis
  • stretch will stretch the item to make it take up the same space as the largest element
  • center will center along the cross axis
  • baseline will align the content of the item with the other item's content along the main axis

Order

  • order can be used to change an item's order inside the flexbox container
  • each item has a default position of 0 and it just lists them in the order they exist in the html
  • so giving an item -1 will put it at the start, and 1 will put it at the end
  • these numbers can go up or down if you want to move multiple items

Flex-Grow

  • flex grow will make an item grow to take up as much as it can along the main axis
  • all flex items default to 0
  • setting a single item to 1 will make it grow to take up all remaining space it can
  • it's relative, so it you have an item at 1, and another at 2, 2 will take up 2x the space available
  • items will not wrap due to flex-grow, they will only take up available space on their main axis

Flex-Shrink

  • determines if an item is allowed to shrink or not
  • each items defaults to 1 meaning yes it is allowed
  • if you set it to 0, the item will not shrink as the viewport shrinks

Flex-Basis

  • determines how much of the container to take up
  • each item start at auto, but you can set it to for example 20% on an item
  • this will make that item take up 20% of the space it can
  • If flex-wrap is turned on, flex-basis can cause items to wrap

Flex

  • is a shorthand for flex-grow, flex-shrink and flex-basis
  • you can use flex: 0 1 auto; for example to define all 3 in one line in that order
CSS Grid

Grid Template Rows/Columns/Areas

  • The rows property is used to provide the height is each grid row :: grid-template-rows: repeat(2, 150px);
  • The columns property is used to provide the width of each column :: grid-template-columns: repeat(3, 1fr);
  • Areas is for naming sections declated grid :: grid-template-areas: "green brown blue" "violet orange pink";
  • The above would name the 3 grid cells on row 1, then name the 3 on row 2

FR

  • fr stands for fractional unit, and it can be used to divide up the entirety of the available grid area
  • If you were to give 3 columns each 1fr, you would have 3 even colunm widths inside their parent
  • If you gave 2 columns each 1fr and the 3rd column 2fr, the 3rd column would be 2x the width of the others
  • So the size of 1fr is relative to the parent container and how many fr other grid items are given

Naming Grid Rows/Columns

  • You can name rows/columns when you declade them for example...
  • grid-template-rows: [header-begin] 1fr [header-end] 1fr
  • Would create 2rows, with the 1st named header-begin and the 2nd header-end
  • This will name the grid lines to make for more semantic usage later on
  • If your grid will need to be responsive you'd need to redefine names in a media query

Auto Fill/Fit

  • auto-fill can be used to fill out remaining grid space if needed creating new rows/columns implicitly
  • auto-fit will instead collapse implicitly created grid spaces if they are empty
  • ex. :: grid-template-columns: repeat(auto-fit, minmax(100px, 1fr)) if used in a 1000px wide container
  • elements would fill the entire space, being 1fr max width, and 100px min width

Grid Row/Column Gap

  • This property is used to define the size of the gap or gutter between each row or column respectively
  • Using grid-gap: 20px 40px is a shorthand way to set both. It sets row then column
  • A single value applies to both so grid-gap: 20px would set both to 20px
  • Gap can be set individually using grid-row-gap and grid-column-gap respectively

Grid Row/Column Start/End

  • This property determines where a grid item starts based off grid lines
  • If it starts at row 2 then that is the row it will be placed in
  • If the item starts at column 3 then it will be so
  • In this example the item would be in the 2nd row in the 3rd column
  • This can be set on grid items using grid-row: and grid-column: respectively
  • If an item needs to span multiple rows or columns you can do grid-row: 2/7 for example
  • That would start the item at grid row 2 and end ad the end of grid row 6
  • To make an item just extend to the very end of the grid you can use -1 as the end point

Grid Auto Rows/Columns/Flow

  • This property is used to size implicit grid elements
  • ie elements not defined in the template, but exist in your html and are implicitly added due to size
  • Flow is how extra elements are implicitly added, it's set to row by default
  • If you set it to column then all implicitly added elements will add new columns instead of rows
  • The flow setting will also change how items are ordered if you do not set them explicitly
  • By default items in your html will fill the grid in the same order as they exist in the html
  • If flow is set to column it will move top to bottom then to the next column

Align Items and Justify Items

  • align-items aligns grid elements along the column axis
  • The default is stretch, but there is also start, end, and center
  • justify-items aligns grid elements along the row axis
  • the default is stretch, but there is also start, end, and center
  • you can also use justify-self and align-self to position a single item
